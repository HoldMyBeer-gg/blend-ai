  <action>
    Create .github/workflows/test-bpy51.yml with this exact content:

    ```yaml
    name: Handler Tests (bpy 5.1)

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      handler-tests:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - name: Set up Python 3.13
            uses: actions/setup-python@v5
            with:
              python-version: "3.13"

          - name: Install dependencies
            run: |
              python -m pip install --upgrade pip
              pip install bpy==5.1.0
              pip install pytest pytest-cov

          - name: Run handler tests
            run: pytest tests/test_addon/test_handlers/ -v --tb=short

          - name: Run full test suite
            run: |
              pip install -e ".[dev]"
              pytest tests/ -v --tb=short
    ```

    Key design decisions:
    - Python 3.13 ONLY — bpy==5.1.0 requires Python ==3.13.* exactly (per research Pitfall 3)
    - Separate from pylint.yml — that workflow uses Python 3.8/3.9/3.10 which cannot install bpy==5.1.0
    - Triggers on push to main and PRs to main
    - Two test steps: first runs handler tests specifically (fast feedback), then runs full suite
    - Uses actions/setup-python@v5 (not v3 like pylint.yml)
    - Installs bpy==5.1.0 via pip (real Blender Python module, not fake-bpy-module)
    - Uses pytest-cov for optional coverage reporting
  </action>
